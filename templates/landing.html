tracepath.world is deployed on Amazon AWS (Seatle, US). After tracing a few popular websites (based on alexa.com) from different continents, it was observed that the website contents were not served from the respective countries/continents in all cases. This is due to the fact that, CDNs use the location of the DNS to redirect the request to the closest server. In our experiments the contents for most of the websites were severved from the US. About

This is a visual traceroute application that displays the path taken by the Internet packets across the IP networks to reach a given destination. However, the traceroute result may be influenced by firewalls, Network Address Translation and IP tunnels.
This application is deployed on Amazon AWS located in the US, So each request would originate from the US. This tool uses third-party publicly accessible IP geo-location services, it could be possible that the geo-location data in those services is incorrect and thereof the visual representation of traceroute will be incorrect as well.

For educational purposes only. This project is done as part of CSE 534 Fundamentals of Computer Networks course.
